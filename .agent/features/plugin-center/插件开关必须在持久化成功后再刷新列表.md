# 插件开关必须在持久化成功后再刷新列表

## 问题

内置插件详情面板修改启用状态后，会通过 `catrace:plugin-enabled-changed` 通知 `Plugins.vue` 重新读取状态。若面板在自动保存开始前就发送事件，列表会立即从 Store 或 SQLite 读到旧值，导致右侧开关已关闭，但左侧状态点和排序仍保持原状态。

## 约定

- 面板内的开关值可以立即更新，提供即时视觉反馈。
- `catrace:plugin-enabled-changed` 必须在持久化成功后发送。
- 保存失败时不发送刷新事件；`useAutoSavedSetting` 负责把面板值回滚到最近一次成功保存的值。
- 使用防抖自动保存的插件，需要记录此次保存是否包含主开关变化，避免编辑普通配置时也触发列表重读和重新排序。
- 直接等待后端写入的面板，例如 Agent 插件，可以在 await 成功后直接发送事件。

## 涉及文件

- `src/components/plugins/RestPluginPanel.vue` — Store 写入成功后发送事件。
- `src/components/plugins/TimerPluginPanel.vue` — 防抖保存成功且本轮包含启用状态变化时发送事件。
- `src/components/plugins/AgentPluginPanel.vue` — 后端命令成功后发送事件，已有正确顺序。
- `src/views/Plugins.vue` — 收到事件后重新读取三种内置插件的权威状态。
