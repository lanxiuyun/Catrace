# 插件状态排序和统一详情顶栏实现约定

## 左侧列表

插件列表先按 `enabled` 排序，启用的插件在前；同一启用状态内按显示名称升序排序。

```ts
[...builtins, ...externals].sort((a, b) => {
  if (a.enabled !== b.enabled) return a.enabled ? -1 : 1
  return a.name.localeCompare(b.name, undefined, { sensitivity: 'base' })
})
```

内置插件的启用状态不能写死为 `true`：

- 久坐提醒读取 Store key `plugin_rest_ui_enabled`。
- 定时提醒读取 `getTimerSettings().enabled`。
- Agent 通知读取 `getAgentNotificationEnabled()`。
- 外部插件使用后端返回的 `ExternalPluginInfo.enabled`。

插件详情开关修改后，面板通过 `window` 发送 `catrace:plugin-enabled-changed`，`Plugins.vue` 监听后重新读取状态，因此列表会立即重新排序。组件销毁时必须移除监听器。

## 右侧详情面板

内置插件面板顶部统一只放：

- 插件图标
- 插件名称
- 插件描述
- 启用/关闭开关（仅 switch，不再显示「插件状态」文字标签）

外部插件详情顶栏同理，只保留开关。

## 布局约定

`.plugin-rail` 使用 `width: 15rem; min-width: 15rem; flex: 0 0 15rem` 固定左栏宽度，窗口变窄时也不允许被压缩。
