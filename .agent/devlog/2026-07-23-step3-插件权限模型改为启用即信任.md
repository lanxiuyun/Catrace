# 2026-07-23 Step3 插件权限模型改为启用即信任

## Session goal

复核 Step 3 的 manifest 权限门闩是否符合 Catrace“本地插件自由优先”的产品方向，并把结论更新到计划真源。

## Completed

- 对比 VS Code、uTools、Rubick 的桌面插件信任方式：安装/启用是主要信任动作，普通宿主能力通常不采用浏览器扩展式逐项授权。
- 决定采用 Trusted Local Plugin Model：用户手动安装并启用插件后，可直接调用已公开的普通宿主 API。
- `permissions` 降级为可选能力说明元数据，不再作为普通 invoke 的运行时硬门闩。
- 明确自由调用不等于取消边界：身份、启用状态、Event 所有权、私有命名空间、输入大小和防失控仍由 Rust 强制。
- 更新 Step 3 路线图的 M11.1、宿主能力、风险、完成定义和 M12 固定边界。
- 更新 manifest 当前下一步，指向移除权限硬门闩和补齐隔离/防失控。

## Next

- 为跨插件冒充、禁用后残留调用、输入边界和 publish 防失控补测试。
