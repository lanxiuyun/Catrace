# 2026-07-24 Step3 M11.1 启用即信任与异常插件观测落地

## Session goal

完成 Step 3 M11.1 的代码闭环：移除普通插件能力的 manifest 权限硬门闩和隐藏资源硬限制，同时保留启用状态、调用身份、Event 所有权及 Storage 命名空间边界，并记录异常活跃插件。

## Completed

- 插件 manifest、Rust/TypeScript 插件类型和 demo manifest 均删除 `permissions` 字段；能力是否开放由宿主 API 与 Rust 固定边界决定。
- 后台命令仍只从 `plugin-bg-<id>` 窗口 label 推导身份，并统一确认插件已安装、有效且启用。
- Toast 卡片剪贴板调用仍要求 `reminder-toast` 窗口、目标 Event source 与显式 plugin id 一致且插件已启用。
- Event 发布继续复用 `allows_event`，强制 source、kind/event_type 白名单和保留 kind 边界。
- 新增按插件独立的资源活动观测：连续高内存、连续 Storage 磁盘写入、单次大数据或 60 秒内刷事件时记录带 plugin id 的 warning，并在 Plugins 页面实时显示“异常” Tag；调用继续执行，不限流、不丢弃。
- Event shape 只保留协议必需字段、action id 唯一性和 progress 合法性；不限制 title、body、actions 数量或 payload 大小。
- 删除 Storage value、剪贴板文本、日志 message/data 的大小和非空限制；未知日志 level 仍按 info 记录并保留原 level。
- 单元测试覆盖连续高内存、连续磁盘写入、单次大数据与刷事件只告警不拦截，以及 Event 大内容不被大小阈值拒绝和 Storage namespace 边界。
