# 插件配置和运行数据必须分开存储

插件持久化分成两个明确接口，避免把可迁移设置、机器运行状态和业务历史混在 SQLite 的通用 `settings` 表中。

## 存储边界

| 接口 | 位置 | 用途 | 示例 |
|---|---|---|---|
| `plugin_config` | Tauri Store `settings.json` | 用户可理解、可备份、可迁移到其他电脑的插件设置 | 插件启用状态、提醒规则、文案、Agent 声音设置 |
| `plugin_storage` | SQLite `plugin_storage` 表 | 机器相关运行状态、插件持久化业务数据 | timer 上次触发时间、daily 去重键、外部插件运行数据 |

Store key 统一为 `plugin_config:<plugin-id>`，值是该插件的 JSON 对象。Rust 后台通过 `tauri_plugin_store::StoreExt` 读写同一份 Store，不为后台再复制一份 SQLite 配置。

SQLite 表固定为：

```sql
plugin_storage (
  plugin_id TEXT NOT NULL,
  key TEXT NOT NULL,
  value TEXT NOT NULL,
  PRIMARY KEY (plugin_id, key)
)
```

## 内置插件落点

- `rest`：启用状态、工作/休息时长、推迟时长、自定义文案都在 `plugin_config:rest`。
- `timer`：启用状态和规则定义在 `plugin_config:timer`；`last_fired_at`、`last_daily_keys` 写入 `plugin_storage(timer, runtime)`。
- `agent`：启用状态、事件显示策略、声音模式/路径/音量在 `plugin_config:agent`。
- 外部插件：启用状态和 `plugin_config_get/set` 写 Store；`plugin_storage_get/set` 写 SQLite，并由后台 WebView 身份绑定插件 id。

## 已删除的旧边界

独立喝水提醒、护眼提醒及 `water_records` 已删除。喝水/护眼现在只是定时提醒插件的预设，不再拥有独立后端状态、设置卡、Toast 卡或 Dashboard 统计组件。

## 新增配置时的判断

1. 用户换电脑后希望继续使用吗？是 → `plugin_config`。
2. 是否是触发游标、缓存、去重状态或机器产生的业务记录？是 → `plugin_storage`。
3. 不要再创建 `plugin_storage:<id>:<key>` 形式的通用 settings key。
4. 简单插件配置不要进入 SQLite `settings` 表。
