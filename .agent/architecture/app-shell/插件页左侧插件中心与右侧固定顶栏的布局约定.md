# 插件页左侧插件中心与右侧固定顶栏的布局约定

## 页面结构

插件页不是普通的整页纵向滚动，而是在应用全局顶栏下方独立占满工作区：

```text
Plugins.vue
├── plugin-rail（15rem）
│   ├── rail-header（插件中心、数量、刷新）
│   ├── rail-search（本地筛选）
│   └── OverlayScrollbar
│       ├── 插件列表与状态圆点
│       └── 探索更多插件
└── plugin-main
    └── 各插件 Panel
        ├── 固定 panel-header
        └── OverlayScrollbar
            └── panel-body
```

## 左侧插件中心

- 左栏宽度保持 `15rem`，窄窗口可收窄到 `13rem`，不切换成上下布局。
- 标题区只放插件中心、当前插件数量和刷新按钮，避免堆叠与插件管理无关的信息。
- 搜索只在前端过滤已加载插件，匹配名称、说明和插件 ID，不触发后端扫描。
- 状态圆点使用现有注册和启用状态：正常为绿色、未启用或未注册为灰色、加载错误为红色。
- “探索更多插件”当前复用打开本地插件目录的能力，不伪造尚不存在的插件市场入口。

## 右侧插件面板

`RestPluginPanel.vue`、`TimerPluginPanel.vue`、`AgentPluginPanel.vue` 采用相同的滚动边界：顶部标题和总开关固定，只有下面的配置内容滚动。新增内置插件面板也应保持这一结构，避免整个页面滚动后丢失插件身份和主开关。

## 全局顶栏约定

- 应用全局顶栏高度为 `3rem`，左 padding 固定为 `0.75rem`。
- 一级导航使用紧凑圆角选中态，不随窗口宽度动态扩大 padding。
- 交流群属于应用级辅助信息，放在全局顶栏最右侧的弱强调 Tag 中，不占插件侧栏空间。

## 涉及文件

- `src/App.vue` — 全局顶栏、一级导航与交流群 Tag。
- `src/views/Plugins.vue` — 插件中心侧栏、搜索、状态和左右布局。
- `src/components/plugins/RestPluginPanel.vue` — 久坐插件固定顶栏与内容滚动。
- `src/components/plugins/TimerPluginPanel.vue` — 定时插件固定顶栏与内容滚动。
- `src/components/plugins/AgentPluginPanel.vue` — Agent 插件固定顶栏与内容滚动。
- `src/components/OverlayScrollbar.vue` — 覆盖式滚动实现。
