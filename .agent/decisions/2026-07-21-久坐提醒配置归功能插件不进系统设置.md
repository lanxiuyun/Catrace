# 久坐提醒配置归功能插件，不进系统设置

## 状态

Accepted — 2026-07-21

## 背景

设置页同时存在「系统设置」与「功能插件」。久坐相关配置曾散落在系统设置的通知卡（`NotificationSettingsCard`）与功能插件 `RestPluginPanel` 两侧，导致：

- 用户不知道「多久提醒 / 提醒什么 / 怎么提醒」该去哪改
- 系统设置被塞进产品功能细节，边界模糊

## 决策

**久坐提醒的 when / what / how 全部归属功能插件 `rest`（`RestPluginPanel`）。**

系统设置只保留宿主级能力：

| 系统设置 core | 含义 |
|---------------|------|
| `media` | 视频与音乐（跨功能宿主能力） |
| `signal` | 行为采集 |
| `system` | 语言 / 自启 / 更新 |
| `links` | 相关链接 |

## 归属边界

| 配置 | 归属 | 理由 |
|------|------|------|
| 工作窗口 / 休息阈值 / 再提醒间隔 | Rest 插件 | 久坐插件核心节奏 |
| 提醒模式 toast / fullscreen | Rest 插件 | 久坐如何呈现 |
| 标题 / 正文 | Rest 插件 | 久坐提醒什么 |
| 全屏背景 / 透明度 / 适配 | Rest 插件 | fullscreen 模式细节 |
| 测试通知 | Rest 插件 | 验证本插件配置 |
| 视频与音乐 | 系统设置 media | 宿主媒体能力，不只服务久坐 |
| Signal 采集 | 系统设置 signal | 宿主感知层，非单一插件 |

## 后果

- 删除 `NotificationSettingsCard.vue`；`Settings.vue` core 键不再含 `notification`
- 后端键名保持不变（`reminder_mode` / `reminder_title` / `reminder_body` / fullscreen_*），仅 UI 入口收敛
- 媒体仍在系统设置，避免把宿主能力硬塞进 rest 插件
