# 2026-07-24 插件关闭后列表状态点和排序没有更新

## Symptom

在功能插件详情中关闭久坐提醒或定时提醒后，右侧开关和禁用内容立即更新，但左侧插件列表仍显示绿色状态点，也没有按禁用状态重新排序。

## Root cause

久坐和定时面板使用 `useAutoSavedSetting` 异步保存状态，却在修改本地响应式值后立即发送 `catrace:plugin-enabled-changed`。`Plugins.vue` 收到事件后马上读取 Store/SQLite，此时持久化尚未完成，因此读回旧值。

定时插件还有 500ms 防抖窗口，所以该竞态稳定可复现；久坐插件即使 debounce 为 0，也仍通过定时任务异步保存，同样可能先刷新列表、后写 Store。

## Fix

- 久坐插件把列表刷新事件移动到 Store 保存成功回调。
- 定时插件记录主开关是否发生变化，仅在对应防抖保存成功后发送列表刷新事件。
- 保存失败时清理待通知状态，由自动保存 composable 回滚右侧开关。

## Verification

- `vue-tsc --noEmit` 通过。
- 需要真机确认关闭/启用久坐和定时插件后，左侧状态点和排序在保存完成后同步更新。
