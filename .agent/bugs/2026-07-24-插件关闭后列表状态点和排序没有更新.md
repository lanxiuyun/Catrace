# 2026-07-24 插件关闭后列表状态点和排序没有更新

## Symptom

在功能插件详情中切换启用状态后，右侧开关会变化，但左侧插件列表的状态点和排序要延迟更新；早期实现甚至可能一直保留旧状态。

## Root cause

插件主开关曾与整份插件配置共用 `useAutoSavedSetting`：久坐和定时配置需要等待 500ms 防抖，再经 Rust 写入 Store，列表又在保存完成后重新读取，因此主开关承担了不必要的保存延迟。

不同插件的启用路径也不一致：Agent 立即保存，外部插件等待 Rust 返回后才更新 UI，久坐和定时则走整份配置防抖保存。

## Fix

统一所有插件主开关规则：

1. 点击后立即乐观更新详情开关、左侧状态点和排序。
2. `enabled` 使用专用 Rust 命令立即写入 `plugin_config:<id>.enabled`，不经过其他设置的 500ms 防抖。
3. 保存失败时同时回滚详情开关和左侧列表。
4. 久坐时长、定时规则等其他配置继续使用防抖保存，但保存接口不再覆盖 `enabled`。
5. 外部插件同样先更新本地列表，再执行后台启停和 Store 保存；失败时恢复原插件信息。
